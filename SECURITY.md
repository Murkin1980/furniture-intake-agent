# SECURITY.md — Правила безопасности

## Общие принципы

1. Минимизация данных — собирать только необходимое
2. Шифрование — данные в покое и при передаче
3. Разделение — данные разных компаний не смешиваются
4. Аудит — отслеживать доступ к данным
5. Удаление — возможность удалить данные по запросу

## Какие сообщения хранятся

### Хранятся
- Текстовые сообщения клиента
- Текстовые ответы AI
- Файлы (фото, документы)
- Метаданные (время, тип, статус)

### Не хранятся
- Пароли пользователей (используется OAuth/хеширование)
- Токены API в открытом виде
- Данные кредитных карт
- Данные, не связанные с заявкой

## Кто видит данные

| Роль | Доступ |
|------|--------|
| Владелец компании | Все заявки своей компании |
| Менеджер | Заявки, назначенные ему |
| AI-агент | Заявки в режиме AI_MODE |
| Админ системы | Технический доступ (логи, ошибки) |

## Разделение компаний

```sql
-- Все запросы фильтруются по company_id
SELECT * FROM leads WHERE company_id = $1;
```

- Данные разных компаний физически в одной таблице, но логически разделены
- Нет механизма для просмотра данных другой компании
- API-запросы без company_id отклоняются

### WhatsApp Business API
- Хранятся в переменных окружения
- Не коммитятся в git
- Не логируются

### OpenAI API
- Хранятся в переменных окружения
- Не коммитятся в git
- Не логируются

### Telegram Bot Token
- Хранятся в переменных окружения
- Не коммитятся в git
- Не логируются

### Database
- Пароли в переменных окружения
- Не хранятся в коде
- Регулярная ротация

## Что запрещено логировать

### Нельзя логировать
- Текст сообщений клиентов
- Номера телефонов
- Email-адреса
- API-ключи и токены
- Пароли
- Финансовые данные (бюджет клиента)

### Можно логировать
- ID заявки
- Статус заявки
- Время обработки
- Ошибки системы (без чувствительных данных)
- Метрики (количество заявок, время ответа)

## Правила webhook

### WhatsApp Webhook
- Верификация подписи обязательна
- Использовать HTTPS
- Обрабатывать дублирующие сообщения
- Валидировать формат данных
- Отвечать 200 OK в течение 5 секунд

### Telegram Webhook
- Верификация токена обязательна
- Использовать HTTPS
- Обрабатывать повторные обновления
- Валидировать формат данных

## Удаление данных

### По запросу клиента
- Удалить все сообщения
- Удалить все фото
- Удалить заявку
- Сохранить аудит-лог (без содержимого)

### По запросу компании
- Удалить все данные компании
- Удалить всех пользователей
- Удалить все заявки
- Удалить все сообщения

### Автоматическое удаление
- Сообщения старше 2 лет — архивировать
- Файлы старше 1 года — удалять
- Логи старше 6 месяцев — удалять

## Резервное копирование

### Частота
- Ежедневное автоматическое копирование
- Хранение 30 дней
- Еженедельное полное копирование

### Где хранить
- Отдельное хранилище (не на основном сервере)
- Шифрование при хранении
- Шифрование при передаче

### Восстановление
- Тестирование восстановления раз в квартал
- Время восстановления: не более 4 часов
- Уведомление при ошибке копирования

## Доступ к серверу

### Протоколы
- SSH с ключами (без паролей)
- VPN для доступа к базе данных
- 2FA для всех аккаунтов

### Журнал доступа
- Логирование всех входов
- Уведомление о подозрительной активности
- Автоматическая блокировка при 5 неудачных попытках

## Обновления безопасности

### Регулярно
- Еженедельно: проверка зависимостей на уязвимости
- Ежемесячно: обновление зависимостей
- Ежеквартно: аудит безопасности

### По факту
- Критические уязвимости: исправление в течение 24 часов
- Высокие: исправление в течение 72 часов
- Средние: исправление в течение 1 недели

## Ответственность

### Разработчик
- Следовать правилам безопасности
- Не коммитить секреты
- Использовать шифрование
- Логировать только разрешённые данные

### Компания-клиент
- Хранить свои ключи в безопасности
- Не передавать доступ третьим лицам
- Уведомлять о инцидентах

### Система
- Автоматически блокировать подозрительную активность
- Уведомлять администратора об ошибках
- Не допускать утечку данных
