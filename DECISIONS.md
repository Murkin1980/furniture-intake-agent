### Решение: Технологический стек

**Дата:** 2026-07-15

**Контекст:**
Определение базовых технологий для MVP. Нужно выбрать стек, который:
- Подходит для быстрой разработки
- Легко масштабируется
- Имеет хорошую поддержку
- Не требует сложной инфраструктуры

**Рассмотренные варианты:**
1. FastAPI + PostgreSQL + Redis
2. Django + PostgreSQL
3. FastAPI + SQLite (для начала)
4. Node.js + MongoDB

**Выбранный вариант:**
1. FastAPI + PostgreSQL

**Почему:**
- FastAPI: быстрый, современный, хорошая документация
- PostgreSQL: надёжный, проверенный, хорошо масштабируется
- Redis: не нужен в MVP, добавить позже при необходимости

**Последствия:**
- Нужно установить PostgreSQL
- Нужно изучить SQLAlchemy для работы с БД
- Нужно настроить миграции (Alembic)

**Когда пересмотреть:**
Если потребуется реальная времени обработка сообщений или фоновые задачи

### Решение: Первая категория мебели — кухня

**Дата:** 2026-07-15

**Контекст:**
Нужно определить, с какой категории мебели начать. Все категории одновременно слишком сложно.

**Рассмотренные варианты:**
1. Кухня на заказ
2. Шкафы и гардеробные
3. Все категории сразу

**Выбранный вариант:**
1. Кухня на заказ

**Почему:**
- Кухни — самая частая категория заказов
- Чёткие параметры для сбора (размеры, стиль, фасады, техника)
- Понятный процесс замера
- Высокий средний чек

**Последствия:**
- AI-логика будет специфична для кухонь
- Расширение на другие категории потребует адаптации
- Менеджеры привыкают к формату заявок

**Когда пересмотреть:**
После 3 пилотных компаний, если кухни не покажут эффективность
