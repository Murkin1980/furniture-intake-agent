# Сценарий передачи заявки менеджеру

## Назначение

Этот документ описывает процесс передачи заявки от AI-приёмщика к менеджеру мебельной компании.

---

## Моменты передачи

### 1. Автоматическая передача (все данные собраны)

**Условие:** Все обязательные поля заполнены

**Что происходит:**
1. AI завершает сбор данных
2. Формирует резюме заявки
3. Обновляет статус заявки на `ready_for_manager`
4. Отправляет уведомление в Telegram
5. Отправляет сообщение клиенту

### 2. По запросу клиента

**Условие:** Клиент пишет «Хочу поговорить с менеджером»

**Что происходит:**
1. AI подтверждает запрос
2. Передаёт управление
3. Отправляет уведомление в Telegram
4. Отправляет сообщение клиенту

### 3. Тайм-аут

**Условие:** Клиент не отвечает 24 часа

**Что происходит:**
1. AI отправляет напоминание
2. Если нет ответа ещё 24 часа — помечает как `waiting_for_client`
3. Через 7 дней — автоматически закрывает как `lost`

## Процесс передачи

### Шаг 1: Подготовка данных

```python
def prepare_handoff(lead: Lead, session: IntakeSession) -> dict:
    return {
        "lead_id": lead.id,
        "client_name": lead.name,
```

### Шаг 4: Ожидание действий менеджера

**Варианты действий:**

1. **Взять в работу** — менеджер берёт заявку
2. **Запросить данные** — менеджер просит AI уточнить
3. **Назначить замер** — менеджер назначает выезд
4. **Отметить нецелевая** — заявка закрывается

## Действия менеджера

### Взять в работу

**Что происходит:**
1. Менеджер нажимает кнопку
2. Статус заявки → `manager_review`
3. Режим диалога → `human_mode`
4. AI прекращает самостоятельную отправку
5. Менеджер получает доступ к истории диалога

**Сообщение клиенту (от менеджера):**
```
Здравствуйте, Айгуль!
Меня зовут [Имя], я менеджер [Компания].
Ваша заявку на кухню принята.

Расскажите подробнее о ваших пожеланиях.
```

### Запросить данные

**Что происходит:**
1. Менеджер указывает, каких данных не хватает
2. AI получает задачу уточнить
3. AI отправляет сообщение клиенту
4. Клиент отвечает
5. AI обновляет заявку

**Сообщение клиенту (от AI):**
```
Менеджер попросил уточнить:
— Высоту потолка
— Расположение коммуникаций

Можете указать это сейчас?
```

### Назначить замер

**Что происходит:**
1. Менеджер выбирает дату и время
2. Клиент получает приглашение
3. Заявка обновляется

**Сообщение клиенту:**
```
Замерщик приедет:
📅 [дата]
⏰ [время]
📍 [адрес]

Подтвердите, пожалуйста.
```

### Отметить нецелевая

**Что происходит:**
1. Менеджер указывает причину
2. Заявка закрывается со статусом `lost`
3. Клиент не получает сообщений

**Причины:**
- Не кухня
- Спам
- Конкурент
- Другое

## AI после передачи

### Режим работы в HUMAN_MODE

AI может:
- Формировать черновики сообщений
- Выделять новые данные из ответов клиента
- Напоминать менеджеру о задачах
- Обновлять карточку заявки

AI не может:
- Отправлять сообщения клиенту самостоятельно
- Изменять статус заявки
- Закрывать заявку

### Анализ сообщений клиента

```python
def analyze_client_message(message: str, lead: Lead):
    # Извлечь новые данные
    new_data = extract_data(message)
    
    if new_data:
        # Обновить заявку
        lead.update_data(new_data)
        
        # Уведомить менеджера
        notify_manager_about_update(lead, new_data)
    
    # Определить намерение
    intent = classify_intent(message)
    
    if intent == "question":
        # Предложить черновик ответа
        draft = generate_draft_answer(message, lead)
        notify_manager_about_draft(lead, draft)
    
    if intent == "complaint":
        # Срочно уведомить менеджера
        notify_manager_urgent(lead, message)
```

## Статусы заявки при передаче

```
new
  ↓ (AI начал сбор)
ai_collecting
  ↓ (все данные собраны)
ready_for_manager
  ↓ (менеджер взял в работу)
manager_review
  ↓ (назначен замер)
measurement_scheduled
  ↓ (подготовлен расчёт)
quote_preparation
  ↓ (расчёт отправлен)
quote_sent
  ↓ (клиент согласился)
won
  ↓ (клиент отказался)
lost
```

## Временные рамки

| Этап | Время | Действие |
|------|-------|----------|
| AI сбор данных | 0–10 минут | Автоматически |
| Ожидание менеджера | 0–2 часа | Уведомление |
| Ответ менеджера | 0–24 часа | Контакт с клиентом |
| Напоминание | Через 24 часа | Если нет ответа |
| Закрытие | Через 7 дней | Если нет активности |

## Метрики передачи

| Метрика | Описание | Цель |
|---------|----------|------|
| Время до передачи | От начала до ready_for_manager | < 15 минут |
| Процент автоматической передачи | Без запроса клиента | > 70% |
| Время ответа менеджера | От уведомления до действия | < 2 часа |
| Процент взятых в работу | От переданных до взятых | > 90% |
| Процент дошедших до замера | От взятых до замера | > 50% |
