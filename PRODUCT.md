# PRODUCT.md — Продуктовая логика furniture-intake-agent

## Назначение продукта

AI-приёмщик входящих заявок для небольших мебельных компаний. Принимает обращения клиентов через WhatsApp, собирает необходимые данные, формирует структурированную карточку заявки и передаёт менеджеру.

## Целевая компания

- 2–15 сотрудников
- Изготовление корпусной мебели: кухни, шкафы, гардеробные
- Заявки через WhatsApp, Instagram, сайт и рекомендации
- Не использует полноценную CRM или ведёт заявки вручную
- Отвечает клиентам владелец или один менеджер
- Теряет часть обращений или долго собирает исходные данные

## Основной пользователь

- Владелец мебельной компании
- Менеджер по продажам
- Менеджер по приёму заявок

## Проблема

Клиенты пишут «Сколько стоит кухня?» но не указывают:
- Размеры
- Город
- Тип помещения
- Материалы
- Фотографии
- Бюджет
- Желаемый срок

Менеджер вынужден вручную задавать одни и те же вопросы. Часть клиентов остаётся без ответа.

## Обещание продукта

> AI-приёмщик отвечает на новое обращение, собирает необходимые данные и передаёт менеджеру готовую карточку заявки.

## Основной сценарий (MVP)

```
Новое сообщение клиента
→ определение типа мебели
→ сбор обязательных данных
→ получение фотографий и размеров
→ формирование AI-резюме
→ создание карточки заявки
→ уведомление менеджера
→ передача диалога человеку
```

## Первая категория мебели

Только кухня на заказ.

После подтверждения сценария добавить:
1. шкаф
2. гардеробную
3. прихожую
4. детскую
5. мебель в ванную
6. офисную мебель

## Вопросы для заявки на кухню

AI должен постепенно собрать:
1. Имя клиента
2. Город или район
3. Тип кухни (прямая / угловая / П-образная / с островом / неизвестно)
4. Приблизительные размеры
5. Фото помещения
6. Наличие эскиза или дизайн-проекта
7. Желаемый стиль
8. Материал фасадов
9. Нужна ли встроенная техника
10. Примерный бюджет
11. Желаемый срок
12. Нужен ли выезд замерщика
13. Удобное время для связи

Вопросы задаются по одному или небольшими группами. Не превращать разговор в анкету.

## Режимы управления диалогом

### AI_MODE
- AI приветствует клиента
- Задаёт вопросы
- Принимает ответы
- Собирает данные
- Формирует резюме

### HUMAN_MODE
- Менеджер нажимает «Взять в работу»
- AI прекращает самостоятельную отправку сообщений
- AI может формировать черновики, выделять данные, напоминать
- Сообщение клиенту отправляет человек

## Что НЕ входит в MVP

- Полноценная CRM
- Склад и производство
- Карты раскроя
- Точный автоматический расчёт
- 3D-конструктор
- Мобильное приложение
- Телефония
- Instagram Direct API
- Telegram как клиентский канал
- Мультиканальный inbox
- White-label
- Маркетплейс
- Автоматическая массовая рассылка

## Метрики

### Основная продуктовая метрика

Доля входящих обращений, доведённых до следующего коммерческого шага.

### Коммерческий шаг
- Полная заявка
- Звонок
- Замер
- Предварительный расчёт
- Коммерческое предложение

### Дополнительные метрики
- Время первого ответа
- Процент заявок с заполненными обязательными полями
- Процент клиентов, завершивших AI-опрос
- Среднее время менеджера на заявку
- Количество потерянных заявок
- Процент передачи AI → менеджер
- Процент заявок, дошедших до замера
- Количество ручных исправлений

## Стоп-условия

Разработка останавливается, если:
- Компании не подтверждают проблему
- Клиенты отказываются общаться с AI
- Менеджеру приходится исправлять большинство диалогов
- AI увеличивает время обработки
- Компании не готовы подключать канал
- Нет готовности платить
- Три пилота показывают несовместимые потребности

## Условия добавления новых функций

Каждая новая функция добавляется только после:
1. Подтверждённой необходимости у пилотных компаний
2. Измеримой пользы от существующего функционала
3. Полученной регулярной оплаты
